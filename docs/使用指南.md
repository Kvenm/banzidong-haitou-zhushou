# 海投助手 Next — 使用指南

在**本机**运行的一套简易工作台：从 BOSS 直聘拉取（或模拟）候选、人工确认后再用浏览器自动化协助「开聊 / 投递」。**所有登录凭据与数据仅存于你电脑上的 `data/` 目录**，请自行保管，勿将 `db.json` 上传到公开仓库。

---

## 一、环境与安装

| 要求 | 说明 |
|------|------|
| Node.js | **≥ 20** |
| 操作系统 | macOS / Windows / Linux 均可 |
| 网络 | 真实采集与投递需能访问 BOSS 直聘 |

```bash
cd haitou-assistant-next
npm install
```

---

## 二、启动与访问

```bash
npm start
```

终端会打印监听地址，默认为：

**http://127.0.0.1:4173**

如需指定端口：

```bash
PORT=8080 npm start
```

若用浏览器直接打开本地 HTML 文件（`file://`），页面里的接口请求会指向 `http://127.0.0.1:4173`，因此仍需先启动上面的服务。

### 2.1 前端如何使用（界面怎么操作）

**架构说明（无需单独构建前端）**

- 前端代码在仓库的 **`public/`** 目录：`index.html`、`app.js`（原生 ES Module）、`styles.css`。
- **没有** Vue / React 工程，也**没有** `npm run build`。页面由 **`npm start`** 启动的 Node 服务（`src/server.js`）一并提供静态文件，并与 **`/api/*`** 接口同源，这样浏览器才能正常调用采集、保存配置、投递等接口。
- 服务默认只监听 **`127.0.0.1`**，适合本机使用；请在本机浏览器访问，不要用局域网其他设备访问（除非你自己改监听地址）。

**打开界面的步骤**

1. 在项目根目录执行 `npm start`，看到类似「已启动: http://127.0.0.1:4173」的提示。  
2. 用 **Chrome / Edge / Firefox / Safari** 等打开上述地址。  
3. 左侧为**导航**：数据看板、BOSS 登录态、投递配置、候选确认、运行日志；侧栏底部三个按钮为 **开始采集**、**投递已确认**、**原始风格**（Geek 策略）。  
4. 右侧主区域随导航切换内容；右上角可 **导出数据 / 导入 JSON 备份**。  
5. 在「候选确认」里可筛选状态、批量确认、对单行操作（确认 / 拒绝 / 打开职位等）；投递开始后界面会**定时刷新列表**，把「已确认」更新为「已投递」（与后端写入一致）。

更细的字段与状态含义见下文「四、界面说明」。

---

## 三、推荐工作流程

建议按顺序操作，避免在未保存 Cookie 时发起真实自动化。

1. **投递配置**  
   填写关键词、城市、薪资、排除词、黑名单、招呼语模板等，保存。

2. **BOSS 登录态**  
   导出已登录 BOSS 的 Cookie（JSON），粘贴并保存；可选填写同源 LocalStorage JSON（与 Geek 行为更接近时可配）。  
   也可使用「自动获取 Cookie」：会拉起受控浏览器，你在其中登录后凭据写入本地。

3. **开始采集**  
   左侧「开始采集」根据当前配置写入候选列表（具体是否走真实接口取决于当前项目实现阶段）。

4. **候选确认**  
   对待投职位点「确认」，或「批量确认待确认」。可筛选状态、单行「拒绝 / 恢复 / 标记投递 / 标记回复」。  
   「打开职位」会在新标签打开 BOSS 职位页。

5. **投递**  
   - **投递已确认**：一种自动化策略（半自动 / 常规 Puppeteer 流程）。  
   - **原始风格**：Geek 风格策略（stealth + laodeng + anonymize-ua，先落推荐页再进详情等），**更适合与 BOSS 反爬环境对齐**。  
   两种方式都会在服务端**异步**执行；界面在投递启动后会**自动轮询快照**，成功写入数据库后，对应候选会从「已确认」变为「已投递」。

6. **运行日志**  
   查看采集、登录校验、开聊接口、错误与跳过原因。Geek 投递完成后可在此对照统计。

## 四、界面说明

### 4.1 数据看板

- 展示候选总量与各状态数量、回复率等汇总。  
- 「最近候选」可跳转到候选确认页。

### 4.2 BOSS 登录态

- **BOSS Cookie JSON**：一般为扩展导出的数组，形如 `[{"name":"...","value":"...","domain":".zhipin.com"}, ...]`。  
- **LocalStorage JSON**：可选，需为对象 `{ "key": "value", ... }`，用于注入自动化浏览器。  
- **保存登录态**：写入本地数据库。  
- **清空凭据**：删除已保存内容。  
- **自动获取 Cookie**：打开捕获流程，登录成功后写回本地（具体行为见日志）。

### 4.3 投递配置

| 字段 | 含义 |
|------|------|
| 职位关键词 | 多词可用逗号或换行分隔 |
| 城市 | 多个城市逗号或换行 |
| 最低 / 最高薪资 K | 单位：千 |
| 工作经验 | 如 `3-5年` |
| 每日投递上限 | 单轮投递最多处理条数（与后端 `dailyApplyLimit` 一致） |
| 排除关键词 | 职位标题等包含则过滤 |
| 公司黑名单 | 逗号或换行 |
| 招呼语模板 | 多行随机选用；支持占位 `{职位名称}`、`{公司名称}`、`{城市}` |

保存后立即生效于后续采集与自动化（招呼语会用于 Geek 流程中的弹窗填写）。

### 4.4 候选确认

- **状态下拉**：筛选列表。  
- **批量确认待确认**：将所有「待确认」一键标为「已确认」。  
- **投递**（侧边栏）：只会处理当前列表里**已确认**的条目，顺序与当前列表中的已确认顺序一致（受每日上限截断）。

各状态含义简述：

| 状态 | 说明 |
|------|------|
| 待确认 | 新入库，尚未人工确认 |
| 已确认 | 允许进入投递队列 |
| 待沟通 | 部分流程中的中间态（若使用） |
| 已投递 | 自动化判定开聊成功或视为已开聊 |
| 已回复 | 可手工标记 |
| 已拒绝 | 不再投递 |

### 4.5 运行日志

展示最近约 300 条操作与结果，便于排查 Cookie 失效、验证码、页面结构变化等问题。

## 五、数据与备份

- 主数据文件：**`data/db.json`**（配置、凭据、候选、日志）。  
- 仓库默认 **`.gitignore` 忽略 `data/db.json` 与 `data/captures/`**，避免误提交隐私。  
- **导出数据 / 导入数据**（顶栏）：可导出完整 JSON 备份，或在另一台机器上恢复。

| 命令 | 作用 |

| `npm run capture:job` | 对库中首个「已确认」职位打开详情并探测页面结构，输出到 `data/captures/`（开发调试用） |

## 七、环境变量（可选）

| 变量 | 作用 |
|------|------|
| `PORT` | HTTP 端口，默认 `4173` |
| `HAITOU_PREFER_SYSTEM_CHROME=1` | 优先使用本机已安装的 Google Chrome 启动自动化（见 `launchBossBrowser.mjs`） |
| `PUPPETEER_EXECUTABLE_PATH` | 指定 Chromium/Chrome 可执行文件路径 |
| `HAITOU_CAPTURE_JOB_DETAIL` | Geek 流程中职位详情探测：`first`（默认，仅首岗）、`1`（每岗）、`0`（关闭） |

示例：

```bash
HAITOU_PREFER_SYSTEM_CHROME=1 npm start
```

## 八、依赖与「原始风格」说明

- **Stealth、UA 匿名化、laodeng**：用于降低自动化特征；其中 **laodeng** 以 `vendor/laodeng` 形式随仓库分发（上游为 GeekGeekRun 同名插件）。  
- **原始风格**按钮走的是 Geek 策略脚本（推荐页会话、详情开聊、`friend/add` 响应判断等），与「投递已确认」所用策略可能不同，按需选用。

---

## 九、常见问题

**启动后页面空白或接口失败**  
确认终端里服务已监听，且浏览器访问的是 `http://127.0.0.1:4173`，不要用错的端口。

**投递一直失败 / 跳登录**  
Cookie 过期或失效，重新在真实浏览器登录后导出并保存；必要时配置 LocalStorage。

**候选仍是「已确认」，没有变「已投递」**  
稍等数秒让后台跑完；界面会自动轮询。若长时间不变，看「运行日志」是否报失败；失败项可能仍为「已确认」。

**找不到浏览器**  
安装依赖自带的 Chromium，或设置 `PUPPETEER_EXECUTABLE_PATH` / `HAITOU_PREFER_SYSTEM_CHROME`。
